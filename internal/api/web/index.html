Por que isso existe

Dado público brasileiro muda e desaparece silenciosamente. Registros de listas oficiais de sanções são retirados sem trilha pública. Contratos recebem emendas retroativas. Jornalistas precisam provar "esse dado existia em X data" e hoje dependem de captura de tela — tecnicamente contestável.

Fé Pública oferece uma alternativa: um arquivo externo, reprodutível, verificável por qualquer pessoa, e que continua válido mesmo que este servidor suma amanhã. A prova mora em Bitcoin.

O que garantimos:

  • Integridade — dado + prova reconstroem o hash original.
  • Tempo — ancoragem em Bitcoin estabelece "antes de tal bloco".
  • Independência — verificação não depende deste servidor.
  • Reprodutibilidade — a política de coleta é pública no GitHub.

O que não garantimos: autenticidade da fonte (arquivamos o que a API oficial retorna), completude (pode haver gaps de coleta), nem interpretação (não somos fact-checker).

Como verificar uma prova

Baixe uma prova de qualquer evento e valide offline com o CLI standalone:

# 1. Baixar a prova de um evento
curl -s "https://fepublica.gmowses.cloud/snapshots/1/events/<external_id>/proof" > proof.json

# 2. Compilar o binário de verificação (ou baixar da release)
git clone https://github.com/gmowses/fepublica && cd fepublica && make verify-cli

# 3. Verificar — re-calcula hash, valida Merkle, lista anchors OTS
./bin/fepublica-verify proof.json

A verificação faz três checagens: (1) re-hashing do JSON canônico, (2) validação da prova Merkle contra a raiz declarada, (3) listagem dos receipts OpenTimestamps anexados. Para verificação fim-a-fim até a transação Bitcoin, extraia os receipts e rode ots verify do cliente oficial OpenTimestamps.

API

  • GET /health — health check
  • GET /sources — lista de fontes monitoradas
  • GET /snapshots?source=<id>&limit=<n> — lista de coletas
  • GET /snapshots/{id} — detalhes de uma coleta
  • GET /snapshots/{id}/anchors — ancoragens OpenTimestamps da coleta
  • GET /snapshots/{id}/events/{external_id}/proof — prova completa de um evento individual

Todos os endpoints são públicos, stateless e devolvem JSON. Respeite rate limit do upstream se for consumir em escala.
